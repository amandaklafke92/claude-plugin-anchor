# Contributing

Anchor is early. The most useful contributions are small, concrete, and easy to review.

Good contributions include:

- clearer setup or usage docs
- improvements to the plugin skills
- fake examples that explain the data format
- small fixes to roadmap or product docs

Please avoid:

- committing real journal entries or personal data
- large rewrites without discussion
- adding heavy process or abstractions before the need is proven

## Before Opening A Pull Request

- Keep the change focused.
- Use fictional data only.
- Update docs when behavior changes.
- Explain what changed and why.

## Product Direction

Anchor prioritises user-owned Markdown, low-friction capture, and concrete memory reconstruction. Changes should support one of those goals.
