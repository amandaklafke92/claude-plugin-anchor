# Anchor Roadmap

Anchor's current roadmap follows a Now / Next / Later sequence. The source direction comes primarily from `product/strategy/mvp-roadmap.md`.

## Now: Prove The Workflow

Build and refine the core workflow:

```text
capture -> review -> save
```

This is the foundation. Every input type needs the same basic path before the product gets more complicated.

The first usable version should let someone create a typed entry, review the processed result, and save it as a structured Markdown file.

## Next: Support More Inputs

Once the workflow works, expand capture methods:

- voice note input
- uploaded audio files
- handwritten-note photo input
- transcript/source preservation
- consistent review interface
- automatic storage management

The goal is not to add every possible feature. The goal is to let users capture memories in the format they already have, then move each input through the same simple review and save flow.

## Later: Make The Archive Useful

After there are enough structured entries, add archive intelligence:

- search
- monthly reviews
- yearly summaries
- printed life record support

These features become more valuable once the archive has real volume and consistent structure.

## Product Principle

Anchor should stay lightweight. Add structure when it helps users capture, preserve, or return to memories. Avoid process for its own sake.
